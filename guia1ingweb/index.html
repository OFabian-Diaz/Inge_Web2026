Introducción a la ingeniería web

La ingeniería web se enfoca en el diseño, desarrollo, despliegue y mantenimiento de aplicaciones y sitios web. Combina buenas prácticas de ingeniería de software con tecnologías de la web (frontend, backend, bases de datos, seguridad, rendimiento y experiencia de usuario).

Un buen punto de partida es entender cómo se estandariza la web: el MDN Web Docs reúne documentación confiable para HTML, CSS y JavaScript.

Arquitectura cliente-servidor

En la arquitectura cliente-servidor, el cliente (por ejemplo, un navegador) realiza solicitudes y el servidor responde con recursos (HTML, CSS, JS, datos, etc.). La comunicación suele ocurrir usando el protocolo HTTP/HTTPS.

Ejemplo simple del flujo

  1. El usuario escribe una URL en el navegador (cliente).
  2. El navegador envía una solicitud HTTP al servidor.
  3. El servidor procesa la solicitud y devuelve una respuesta.
  4. El navegador renderiza el HTML/CSS y ejecuta JavaScript.

Para profundizar: guía de HTTP en MDN (HTTP Overview) y conceptos de arquitectura web en web.dev.

Importancia de la semántica y el diseño accesible

Artículo breve para explicar por qué HTML5 semántico ayuda a usuarios y equipos.

Usar etiquetas semánticas (como <header>, <main>, <section>, <article>, <aside> y <footer>) permite que el contenido tenga un significado claro. Esto mejora la lectura del código, el SEO y la compatibilidad con tecnologías de asistencia.

Un sitio accesible considera a personas con diferentes capacidades (lectores de pantalla, navegación por teclado, etc.). Para esto, ayudan prácticas como: títulos en orden (h1, h2, h3), enlaces descriptivos, buen contraste, y atributos ARIA cuando sea necesario.

Referencias confiables: W3C WAI (Introducción a Accesibilidad) y guía de HTML semántico en MDN (Semantics).

Idea clave: semántica = estructura con significado; accesibilidad = inclusión y usabilidad.

Reflexión en equipo

¿Por qué organizar semánticamente facilita el trabajo colaborativo y el mantenimiento?

  • Divide el trabajo: cada integrante puede encargarse de una sección (header, main, aside, footer) sin pisarse.
  • El código se entiende rápido: la estructura “se lee” como un mapa del sitio, incluso sin CSS.
  • Mantenimiento más fácil: actualizar una parte no rompe el resto porque cada bloque está bien delimitado.

Tres ventajas observadas del uso de HTML5 moderno

  1. Semántica integrada: etiquetas claras reemplazan muchos div sin significado.
  2. Mejor accesibilidad base: estructura correcta ayuda a lectores de pantalla y navegación.
  3. Compatibilidad y estándares: validación con el validador HTML5 y soporte amplio en navegadores.
